# MAVLink 协议

MAVLink（Micro Air Vehicle Link，微型飞行器链路）是一种专为资源受限系统和带宽受限链路设计的轻量级二进制遥测协议。它是无人机、地面控制站和机载电脑之间通信的标准协议。

## 数据包结构

每个 MAVLink v2 数据包均采用以下固定结构：

| 字段 | 大小 | 说明 |
|------|------|------|
| `magic` | 1 字节 | 帧起始标志，v2 固定为 `0xFD`，收到此值表示新包开始 |
| `len` | 1 字节 | 有效载荷长度（0–255 字节） |
| `incompat_flags` | 1 字节 | **必须**理解的标志位——若无法识别则必须丢弃该包 |
| `compat_flags` | 1 字节 | 可忽略的标志位——无法识别时仍可继续处理该包 |
| `seq` | 1 字节 | 包序号（0–255 循环递增），用于检测丢包 |
| `sysid` | 1 字节 | 发送方系统 ID（如飞机 = 1，地面站 = 255） |
| `compid` | 1 字节 | 发送方组件 ID（如飞控、摄像头、GPS） |
| `msgid` | 3 字节 | 24 位消息类型 ID，最多支持约 1670 万种消息类型 |
| `payload` | 0–255 字节 | 实际消息数据 |
| `checksum` | 2 字节 | CRC-16/MCRF4XX 完整性校验码 |

### 为什么字段在传输时会被重新排序

MAVLink 在发送前会将有效载荷字段**按大小从大到小排序**，即使 XML 消息定义中的顺序不同。排序使用稳定排序算法，相同大小的字段保持原有的相对顺序不变。

这样可以在不添加任何填充字节的情况下保证天然内存对齐：

```
uint64_t  → 地址 0   ✅ 对齐到 8 字节
uint32_t  → 地址 8   ✅ 对齐到 4 字节
uint16_t  → 地址 12  ✅ 对齐到 2 字节
uint8_t   → 地址 14  ✅ 任意地址均可
uint8_t   → 地址 15  ✅ 任意地址均可
```

若不排序，一个 `uint8_t` 后面紧跟 `uint64_t` 会被迫插入 7 字节的填充——这对本已受限的无线链路来说是纯粹的带宽浪费。

## 两种通信模式

MAVLink 专为**混合网络**设计——同一数据流可能有多个消费者，且不同数据对可靠性的要求也不同。

### 广播 / 主题模式

用于持续性遥测数据：位置、姿态、电池电量、传感器读数等。

- 无 `target_sysid` / `target_compid` 字段，数据包更小
- 无需确认回复——丢失的包会被下一包自然替代
- 任意数量的订阅者可同时接收同一数据流

```
飞机 → [地面站、机载电脑、云端服务器]   （发送一次，所有人接收）
```

### 点对点模式

用于指令下发、任务上传和参数修改。

- 包含目标系统 ID 和组件 ID
- 需要确认回复——超时未收到则重发
- 保证数据可靠送达指定接收方

```
地面站 → 飞机   （需要 ACK 确认，超时重传）
```

| | 广播模式 | 点对点模式 |
|--|---------|----------|
| 目标地址 | 无 | 必须指定 |
| 确认回复 | 不需要 | 需要 |
| 丢包处理 | 可接受 | 必须重传 |
| 适用场景 | 遥测数据流 | 指令、参数、任务 |

## MAVLink 在 PX4 中的实现

在 PX4 中，`mavlink` 模块承担着内部 **uORB** 消息总线与外部 MAVLink 协议之间的桥接工作。

```
内部（uORB）                          外部（MAVLink）
──────────────────────────────────────────────────────
GPS 模块      ─┐
飞控模块      ─┤  uORB  →  [mavlink 模块]  →  MAVLink 数据包  →  地面站
传感器模块    ─┘
```

**发送方向（飞机 → 地面站）：**
1. 订阅 uORB 话题（如 `vehicle_gps_position`、`vehicle_attitude`）
2. 按配置频率将 uORB 数据转换为 MAVLink 消息
3. 通过串口 / UDP 发送出去

**接收方向（地面站 → 飞机）：**
1. 接收 MAVLink 数据包（任务上传、参数设置、模式切换等）
2. 解析后发布到对应的 uORB 话题
3. 相关模块自动响应并执行

### 源码位置

```
PX4-Autopilot/src/modules/mavlink/
├── mavlink_main.cpp        # 模块入口，管理实例和串口
├── mavlink_messages.cpp    # 定义发送哪些消息及其频率
├── mavlink_receiver.cpp    # 处理接收到的 MAVLink 消息
└── mavlink_stream_*.cpp    # 各类遥测数据流的具体实现
```

### 多实例支持

PX4 支持同时运行多个 `mavlink` 实例，每个实例对应不同的物理链路：

| 实例 | 链路 | 带宽 |
|------|------|------|
| 1 | UART → 遥控接收机 | 低 |
| 2 | USB → 地面站电脑 | 高 |
| 3 | 以太网 → 机载电脑 | 最高 |

每个实例可独立配置消息集合和更新频率。

### 常见消息发送频率

| 消息 | 默认频率 |
|------|---------|
| `HEARTBEAT`（心跳包） | 1 Hz |
| `ATTITUDE`（姿态） | 10–50 Hz |
| `GLOBAL_POSITION_INT`（GPS 位置） | 5–10 Hz |
| 传感器数据 | 10–100 Hz |

## 无线电频率（加拿大）

MAVLink 是数据协议，与无线硬件无关。在加拿大（北美地区），常用的遥测链路无线电频率如下：

| 频段 | 常见硬件 | 说明 |
|------|---------|------|
| **915 MHz** | SiK Radio、RFD900 | 北美最常用，传输距离远，穿透力强 |
| **433 MHz** | 部分 SiK Radio | 欧洲更为普遍，加拿大亦可使用 |
| **2.4 GHz** | Wi-Fi、ESP32 | 短距离，带宽较高 |

加拿大无线电频谱由 **ISED（加拿大创新、科学与经济发展部）** 负责监管。使用前请确认所选频段是否属于免许可证的 ISM 频段。
